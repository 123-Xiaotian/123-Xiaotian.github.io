思考题

Thinking 0.1

Untracted.txt中的内容：
位于分支 master

尚无提交

未跟踪的文件:
（使用 “git add <文件>…” 以包含要提交的内容）
README.txt
Untracted.txt

Staged.txt中的内容：
位于分支 master

尚无提交

要提交的变更：
（使用 “git rm –cached <文件>…” 以取消暂存）
新文件： README.txt

Modified.txt中的内容：

位于分支 master
尚未暂存以备提交的变更：
（使用 “git add <文件>…” 更新要提交的内容）
（使用 “git restore <文件>…” 丢弃工作区的改动）
修改： README.txt

结果不一样，第一次add的时候给出的是要提交的变更，第二次给出的是尚未暂存以备提交的变更，第二次必须得再add一次把修改添加到暂存区才能正确提交。

Thinking 0.2

add the file 是 git add
stage the file 是 git add
commit the file 是 git commit

Thinking 0.3

3.1 代码文件 print.c 被错误删除时，应当使用什么命令将其恢复

git restore print.c

3.2 代码文件 print.c 被错误删除后，执行了 git rm print.c 命令，此时应当使用什么命令将其恢复？

git restore –staged print.c
git restore print.c

3.3 无关文件 hello.txt 已经被添加到暂存区时，如何在不删除此文件的前提下将其移出暂存区？

git restore –staged hello.txt

Thinking 0.4

git reset –hard HEAD^ 会将工作区和暂存区全部回退到上一个commit的状态
git reset –hard < hash> 会将工作区和暂存区全部回退到指定commit的状态

Thinking 0.5

输入echo first,first被打印在终端
输入echo second > output.txt ，second被写入output.txt
输入echo third > output.txt ， third被写入output.txt，second被覆盖
输入echo forth >> output.txt ， forth被追加到output.txt

难点分析

本次实验中，exam的难点主要在如果没有提示，如何从文件中提取变量，实际上只需include文件，然后将变量用$()包裹即可

extra部分的难点主要集中在sort()和uniq()的使用,sort的用法是将多个文件连接并按照字典序排序,并用-n -r参数实现数值倒序排序。uniq的用法是将相邻的重复行删除，-c参数可以统计重复行的次数。运用这个方法，并注意文件夹和文件的创建与权限更改，以及路径的制定。

实验体会

实验前一定要注意文件结构！不要覆盖掉课题组给出的Makefile！！！然后注重文件权限，和文件路径以及变量命名和转义符$配合双引号的使用。
